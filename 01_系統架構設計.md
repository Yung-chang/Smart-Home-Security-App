# 智慧家庭資安 APP — 系統架構設計文件
**SmartHome Guardian**
版本：1.0 | 技術棧：Kotlin + Jetpack Compose | Android API 26+

---

## 1. 系統概覽

SmartHome Guardian 是一款以資訊安全為核心的智慧家庭管理應用程式。系統提供：
- 家庭設備統一管理與控制
- 即時異常偵測與警報推播
- 設備存取權限分級管理
- 端對端通訊加密
- 完整操作稽核日誌

## 4. 資安架構設計

### 4.1 身份驗證層
```
認證流程：
用戶啟動 APP
    │
    ▼
生物辨識驗證 (BiometricPrompt API)
    │ 失敗
    ▼
PIN / 密碼備援驗證
    │ 成功
    ▼
JWT Token 取得 (有效期 15 分鐘)
    │
    ▼
Refresh Token 更新 (有效期 7 天，存於 EncryptedSharedPreferences)
    │
    ▼
API / WebSocket / MQTT 授權請求
```

### 4.2 通訊加密
| 通訊協定 | 加密方式 | 說明 |
|---------|---------|------|
| REST API | HTTPS / TLS 1.3 | Certificate Pinning 防中間人攻擊 |
| WebSocket | WSS (TLS 1.3) | 即時設備狀態推播 |
| MQTT | MQTT over TLS 8883 | IoT 設備通訊 |
| 本地儲存 | AES-256-GCM | Room DB 加密，EncryptedSharedPreferences |
| 設備指令 | HMAC-SHA256 簽章 | 防止指令竄改 |

### 4.3 存取控制模型（RBAC）
```
角色定義：
┌──────────────┬──────────────────────────────────────────┐
│ 角色         │ 權限                                      │
├──────────────┼──────────────────────────────────────────┤
│ 管理員       │ 全部設備 + 用戶管理 + 設定 + 稽核日誌    │
│ 家庭成員     │ 指定設備控制 + 查看自己的操作紀錄         │
│ 訪客         │ 特定時段 + 特定設備（唯讀/限制操作）      │
│ 裝置服務帳號 │ 僅 MQTT 發布/訂閱，無 UI 存取             │
└──────────────┴──────────────────────────────────────────┘
```

### 4.4 異常偵測規則
- 短時間內同一設備大量操作（> 10 次/分鐘）→ 自動鎖定
- 非授權時間存取（例如：凌晨 2-6 點門鎖操作）→ 即時警報
- 新 IP / 新裝置首次登入 → 二次驗證
- Token 異地使用偵測 → 強制登出所有裝置
- 設備離線超過閾值 → 推播通知

## 5. 資料模型設計

### 5.1 設備（Device）
```kotlin
data class Device(
    val id: String,               // UUID
    val name: String,             // 顯示名稱
    val type: DeviceType,         // 燈光/門鎖/攝影機/感應器/插座
    val roomId: String,           // 所在房間
    val status: DeviceStatus,     // ONLINE / OFFLINE / ERROR
    val isLocked: Boolean,        // 是否被管理員鎖定
    val lastSeen: Long,           // 最後上線時間戳
    val firmware: String,         // 韌體版本
    val macAddress: String,       // MAC 位址（用於識別）
    val encryptionKey: String,    // 設備通訊金鑰（加密儲存）
    val accessRules: List<AccessRule>
)

enum class DeviceType {
    LIGHT, DOOR_LOCK, CAMERA, SENSOR_MOTION,
    SENSOR_DOOR, OUTLET, THERMOSTAT, ALARM
}
```

### 5.2 安全警報（SecurityAlert）
```kotlin
data class SecurityAlert(
    val id: String,
    val type: AlertType,           // INTRUSION / DEVICE_OFFLINE / AUTH_FAIL / ANOMALY
    val severity: Severity,        // LOW / MEDIUM / HIGH / CRITICAL
    val deviceId: String?,
    val message: String,
    val timestamp: Long,
    val isAcknowledged: Boolean,
    val actionTaken: String?       // 系統自動處理記錄
)
```

### 5.3 稽核日誌（AuditLog）
```kotlin
data class AuditLog(
    val id: String,
    val userId: String,
    val action: String,            // "DEVICE_TOGGLE", "LOGIN", "RULE_CHANGE" 等
    val targetId: String?,         // 操作目標（設備ID/用戶ID）
    val before: String?,           // 操作前狀態（JSON）
    val after: String?,            // 操作後狀態（JSON）
    val ipAddress: String,
    val deviceFingerprint: String, // 手機裝置指紋
    val timestamp: Long,
    val signature: String          // HMAC 防竄改簽章
)
```

## 6. 推播與即時通訊

### 6.1 通知架構
```
設備觸發事件
    │
    ▼
MQTT Broker（Home Gateway）
    │
    ▼
後端 Alert Engine（規則判斷）
    │
    ├─→ WebSocket Push（APP 前台即時顯示）
    │
    └─→ FCM / Firebase（APP 背景推播通知）
```

### 6.2 通知優先級
| 等級 | 範例 | 通知方式 |
|-----|------|---------|
| CRITICAL | 門鎖被強行破解 | 聲音警報 + 振動 + 全螢幕通知 |
| HIGH | 深夜偵測到移動 | 彈出通知 + 振動 |
| MEDIUM | 設備離線 | 靜音通知 |
| LOW | 韌體更新提醒 | 通知抽屜 |
